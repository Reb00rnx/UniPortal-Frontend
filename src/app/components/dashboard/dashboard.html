📋 About the Project

UNI Portal is a full-stack university management system offering a separate interface for students and teachers. It handles user identity, course management, grade tracking, schedule planning, and consultation scheduling — all secured with JWT-based authentication.


UNI Portal is my first full-stack project, built entirely for learning and portfolio purposes. It is not a commercial product and is not intended for production use — the primary goal was to gain hands-on experience with Java, Spring Boot, and Angular by building something real from scratch. The application is a proof-of-concept university management system. Some features may be incomplete or not production-ready.

Backend: Spring Boot 3.3 · Java 21 · Postgresql · REST API at http://localhost:8080  |  Frontend: Angular 17+ · Standalone Components

🎓 Student Module ← YOU

👤 Profile /student/profile
  • Displays user data fetched from GET /api/users/user/{id}
  • Shows per-course grade averages (calculated client-side from all grades)
  • Lists the 5 nearest upcoming schedule events
📚 Courses /student/courses
  • Lists all courses from GET /api/courses/all
  • Shows enrolment status — highlights courses the student already belongs to
  • Enrol button → PATCH /api/courses/{courseId}/enroll/{studentId}
🔍 Course Detail /student/courses/:id
  • Loads full course info: name, description, teacher, students, modules
  • Displays ordered module list (read-only for students)
  • Buttons: Ask AI (Gemini) & Forum (StackOverflow) — external links
📊 Grades /student/grades
  • Fetches all grades via GET /api/grade/student/{id}
  • Groups grades by course name (client-side reduce)
  • Click on a grade → modal with full details (value, description)
📅 Schedule /student/schedule
  • FullCalendar (dayGridMonth view) — interactive monthly calendar
  • Click on a date → modal to add a named event
  • Events saved via POST /api/schedule/{userId}/add
  • Loaded via GET /api/schedule/{userId}/all

👨‍🏫 Teacher Module ← YOU

👤 Profile /teacher/profile
  • Displays teacher data + academic title + department
  • Shows list of assigned courses with student count
  • Upcoming events panel (next 5 from schedule)
  • Consultation management: add/edit day, time, room
  • API: POST /api/consultations · PATCH /api/consultations/update/{id}
🏫 My Classes /teacher/my-classes
  • Lists teacher's own courses via GET /api/courses/teacher/{teacherId}
  • Modal to create a new course → POST /api/courses
  • Click on course → navigates to Course Detail view
🔍 Course Detail /teacher/courses/:id
  • Same component as student but teacher-mode unlocks extra controls
  • Add module form → POST /api/courses/{courseId}/modules
  • Delete module → DELETE /api/courses/modules/{moduleId}
📝 Gradebook /teacher/gradebook?id={courseId}
  • Loads course + all enrolled students
  • Per-student grade table with clickable pills (view/delete modal)
  • Add grade modal: value (5.0→2.0) + description field
  • Values mapped: 5.0→FIVE, 4.5→FOUR_HALF …
  • API: POST /api/grade · DELETE /api/grade/{id}
📅 Schedule /teacher/schedule
  • Same FullCalendar component as student — shared between roles
  • Events are user-scoped (by userId from JWT)

🔓 Public & Shared Pages

🔑 Login /login
  • POST /api/auth/authenticate → JWT saved to localStorage
  • Role decoded from token → redirect to /student/dashboard or /teacher/dashboard
📝 Register /register
  • POST /api/auth/register/student or /register/teacher
📜 Privacy Policy & Contact /student/privacy · /teacher/privacy
  • Static pages, available from footer — also exist as public routes /privacyP and /contactP (no auth required)
